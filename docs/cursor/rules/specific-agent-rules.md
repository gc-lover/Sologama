# Специфические правила агентов для разных задач

## Правила для Game Design агента

### Создание механик
1. **Анализ баланса**: Перед реализацией провести анализ влияния на экономику игры
2. **Тестируемость**: Каждая механика должна иметь четкие метрики для тестирования
3. **Скейлабилити**: Механика должна работать для 1000+ одновременных игроков
4. **Монетизация**: Учитывать влияние на донат и фритуплей баланс

### Дизайн персонажей
1. **Лор**: Каждый персонаж должен иметь backstory и мотивацию
2. **Визуальная идентичность**: Уникальный дизайн в аниме-стиле
3. **Геймплей роль**: Четкое позиционирование в команде/соло игре
4. **Прогрессия**: Логичная эволюционная линия

## Правила для Technical агента

### Архитектура кода
1. **Модульность**: Каждый компонент должен быть независимым
2. **Расширяемость**: Легко добавлять новые фичи без переписывания
3. **Производительность**: Оптимизация с самого начала
4. **Тестируемость**: Возможность unit/integration тестирования

### Unreal Engine специфика
1. **UCLASS/USTRUCT**: Правильное использование UE макросов
2. **GC**: Учитывать garbage collection в дизайне
3. **Networking**: Все multiplayer логика через репликацию
4. **Assets**: Использовать Asset Manager для загрузки

## Правила для AI/ML агента

### Процедурная генерация
1. **Сид система**: Все генерации должны быть воспроизводимы
2. **Баланс**: Генерируемый контент должен соответствовать дизайну
3. **Производительность**: Генерация не должна блокировать основной поток
4. **Вариативность**: Достаточное разнообразие без повторений

### ИИ поведения
1. **Реалистичность**: Поведение должно быть предсказуемым но не читерским
2. **Скейлинг**: ИИ должен адаптироваться к уровню игрока
3. **Производительность**: Минимальное влияние на FPS
4. **Баланс**: Не слишком легкий и не слишком сложный

## Правила для UI/UX агента

### Дизайн интерфейса
1. **Аниме стиль**: Соответствие визуальной концепции
2. **Адаптивность**: Работа на всех целевых платформах
3. **Доступность**: Учитывать цветовую слепоту и другие особенности
4. **Локализация**: Поддержка текстов на разных языках

### Игровые экраны
1. **HUD**: Минималистичный но информативный
2. **Меню**: Интуитивная навигация
3. **Инвентарь**: Логичная организация предметов
4. **Настройки**: Полный контроль над геймплеем

## Правила для Audio агента

### Звуковое оформление
1. **Иммерсивность**: Звук должен погружать в мир игры
2. **Баланс**: Не перекрывать диалоги и важные звуки
3. **Динамика**: Адаптация под действия игрока
4. **Оптимизация**: Минимальный размер файлов

### Музыка
1. **Атмосфера**: Соответствие настроению уровней
2. **Динамика**: Изменение под события
3. **Цикличность**: Бесшовное зацикливание
4. **Адаптивность**: Реакция на действия игрока

## Правила для QA агента

### Тестирование
1. **Покрытие**: Тесты должны покрывать все кейсы
2. **Репродуцируемость**: Баги должны иметь четкие шаги воспроизведения
3. **Приоритизация**: Критические баги имеют высший приоритет
4. **Регрессия**: Проверка что фиксы не ломают существующее

### Автоматизация
1. **CI/CD**: Все тесты в пайплайне
2. **Smoke tests**: Быстрая проверка основных функций
3. **Performance tests**: Мониторинг производительности
4. **Compatibility**: Тесты на разных платформах

## Правила для DevOps агента

### Инфраструктура
1. **Масштабируемость**: Возможность роста до 10k+ игроков
2. **Надежность**: 99.9% uptime для сервисов
3. **Безопасность**: Защита от всех типов атак
4. **Мониторинг**: Полная видимость всех систем

### Деплоймент
1. **Zero-downtime**: Обновления без остановки сервиса
2. **Rollback**: Быстрая отмена неудачных обновлений
3. **Staging**: Полное тестирование перед продакшеном
4. **Automation**: Все процессы автоматизированы

## Правила для Community Manager агента

### Коммуникация
1. **Прозрачность**: Открытость о разработке и проблемах
2. **Реактивность**: Быстрый ответ на вопросы сообщества
3. **Консистентность**: Единый стиль коммуникации
4. **Эмпатия**: Понимание проблем игроков

### Контент
1. **Регулярность**: Постоянный выпуск обновлений
2. **Качество**: Высокий уровень создаваемого контента
3. **Вовлеченность**: Активное взаимодействие с игроками
4. **Аналитика**: Отслеживание вовлеченности и фидбека
